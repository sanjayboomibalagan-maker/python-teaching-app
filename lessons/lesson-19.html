2 Generators (Yield magic)
S

Real-Life Example

Story
Water Tap vs Water Tank

Nee 1000 liters tanni oru tank-la nirappi vachikuta memory romba edukkun. Ithu List maari. Aana Generator thaan water tap maari. Unakku eppo tanni venumo, appo tap thirandha onnu onna vandhute irukkum. Ellathaiyum orey nerathula nirapi vachukaama, thevaiyappo matum create panni 'yield' pannum. Memory romba save aagum!

Python 
# Generator function
def my_generator():
    yield 1
    yield 2
    yield 3

gen = my_generator()
print(next(gen)) # Output: 1

Quick Quiz – 🔥 Advanced Concepts

Decorators and Yield logic catch pannitiga-nu check pannalam!

1. Generator-la return-ku bathula enna keyword use pannanum?

return
yield

Explanation: yield keyword thaan function execution state-a save panni values-a one-by-one tarrum.

2. Generators-oda main advantage enna?

Run speed
Memory efficiency
Explanation: Massive data processing panrappo pathi pathiya generator create panrathala memory useage romba kammi.

3. Decorator-ku ulla ezhuthura temporary function-ku enna peyar?

inner_func
wrapper
Explanation: Usually wrapper nu per vappom, athan main function-a wrap panni extra logic add pannum.

4. Generator-la irundhu adutha value-a edukka enna function use aagum?

next()
follow()
Explanation: next(gen) step-by-step-a output kudukkum.

5. Oru function-ku mela neraya decorators mela mela (@dec1, @dec2) apply panna mudiyuma?

Explanation: Ithu **Chaining Decorators** nu solluvom. Multiple wrappers combine panna mudiyum.
