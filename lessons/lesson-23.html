Story
Hotel Waiter (The Messenger)

Nee oru hotel-ku pora. Nee thaan "Client" (Your program). Kitchen thaan "Server" (API). Nee kitchen-kulla direct-a poi samayakaara kitta kekka mudiyathu. So, oru "Waiter" (API) vandhu un menu-a (Request) kitchen-kku kondu povaaru, apram kitchen-la irundhu saapada (Response/Data) unakku kondu varuvaaru. Ithu thaan API logic.

💡 Quick Concept Check

Python-la popular-ana HTTP requests library peyar enna?

import
2 GET and JSON
C

Concept

Exchange
  • GET: Data-va server kitta irundhu "vaangura" method.
  • JSON: Data-va oru structure-la anupra format (Looks like a Python Dictionary).
  • status_code: 200 na Success, 404 na Not Found.
Python 
import requests

url = "https://jsonplaceholder.typicode.com/todos/1"
response = requests.get(url)

if response.status_code == 200:
    data = response.json()
    print(data['title'])

Quick Quiz – API Mastery

Internet data handling purinjiducha-nu check pannalam!

1. API request success-nu solra standard status code enna?

404
200
500
Explanation: **200 OK** thaan internet protocol-la success signal.

2. API-la irundhu varra data format-a Python dictionary-a maatha enna method use aagum?

.text()
.json()
Explanation: **.json()** method raw response string-a Python dict-a parse pannanum.

3. Data-va server kitta "Request" panni vaanga endha HTTP method use pannuvom?

GET
POST
Explanation: **GET** method thaan reading/fetching-ku thevai padum.

4. JSON data structure logic-padi Python-la irukura entha data type maari irukkum?

List
Dictionary
Explanation: JSON structured key-value pairs-a irukkum, andha format **Dictionary** maariyae irukkum.

5. URL-kulla extra information (API key maari) anuppa requests.get-la enna argument use pannuvom?

params
data
Explanation: **params** (Parameters) thaan URL-oda queries (e.g., ?key=value) anuppa correct choice.
